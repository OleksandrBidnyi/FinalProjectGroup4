1. Исходя из задания, нашей командой біло создано три следующие сущности : User, Hotel и Room.

2. Также, в программе присутствует Интерфейс DAO и соответствующие методы для работы с базами данных, и его имплементации: RoomDAO, HotelDAO и RoomDAO.

3. Базы данных, которые мы используем в программе, хранятся в текстовых файлах в пакете dataBase pakage. Программа считывает их и сохраняет полученные данные благодаря имплементациям DAO.  

3. При имплементациях DAO используется синглтон, во избежание повторного считывания базы и одновременного существования нескольких однотипных коллекций.

4. Базы данных, существующих в программе сущностей, являются связанными, поэтому при удалении, добавлении и прочих методах, влияющих на смену информации в базах данных, производятся синхронные изменения. 

5. Интефейс ControllerInterface имеет необходимые методы для функционирования программы, а его имплементация, представлена реализацией класса Controller. 

5. Консольные классы UserVsAdmin и Admin сделаны для упрощения управления функционалом программы. Важно учитывать факт, что наиболее широкий спектр функционала по работе с данными имеет Admin.

6.Функционал метода console() в классе UserVsAdmin:
1-search rooms,
2- search hotels by name, 
3- search hotel by city.
Authorize for reservation: 
4-enter, 5- registration.
0- finish work

Также важно отметить, что функционал админситратора вызывается с помощью входа в качестве администратора:
admin/0000

7. Функционал консольного метода root() в классе Admin:
1- find user by name, 
2 - users activity(booked rooms, 
3- delete users,
4- find hotels by name, 
5 - find hotels by city, 
6- edit hotels, 
7- add hotels, 
8- delete hotels,
9 - find rooms, 
10- edit rooms, 
11- add rooms, 
12- delete rooms,
0 - exit administrator mode

8. Для запуска программы мы создали соотвествующий класс Start.
